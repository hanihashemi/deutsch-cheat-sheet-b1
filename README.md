# telc Deutsch B1 · Cheat Sheet

A single-page, self-contained study sheet for the **telc Deutsch B1 Prüfung**. Everything lives in one `index.html` — no build step, no dependencies, works offline, and prints cleanly.

**Live:** https://hanihashemi.github.io/deutsch-cheat-sheet-b1/

## Open it locally

Just open `index.html` in any browser:

```bash
open index.html        # macOS
```

## What's inside

**🎓 telc B1 Prüfung**
- Prüfungsüberblick — written & oral parts, timing, ~60 % to pass
- Brief schreiben (Schriftlicher Ausdruck) — letter layout, formal vs. informal, Redemittel by function
- Meinung & Sprechen — opinion, agreeing/disagreeing, structuring, oral Teil 1–3 phrases

**Fälle & Präpositionen** — Wechselpräpositionen (with picture grid), Dativ + Akkusativ, verbs + fixed prepositions, `wo(r)-`/`da(r)-`, reflexive verbs, adjective endings

**Satzbau & Konjunktionen** — word order / verb position (diagram), wann·wenn·denn·dann·weil, als, dass, trotzdem/obwohl

**Konstruktionen & Redemittel** — Lust haben, Was für …, Konditional II, making/accepting/declining suggestions (oral Teil 3), confirming & reacting

**Extras** — hin-/her- directions (diagram), frequency adverbs, `sein` in four tenses

## Visualizations

Original, theme-aware **SVG/CSS diagrams** (no external images, offline-safe):
- 🧭 9 hand-drawn preposition scenes (ball + box for an/auf/in/über/…)
- 🧩 verb-position diagram (Hauptsatz vs. Nebensatz)
- ↕️ hin-/her- direction figure
- 🎓 exam-structure panels, ✍️ letter template, 🎨 adjective-ending decision flow, 📊 frequency bars

## Features

- 🔎 **Live search** — filters rows *and* text inside diagrams/panels (press <kbd>/</kbd> to focus, <kbd>Esc</kbd> to clear)
- 🌓 **Dark mode** — toggle in the toolbar, remembers your choice
- 🧭 **Grouped jump navigation** with active-section highlighting
- 🎨 **Colour-coded cases** — Nominativ, Akkusativ, Dativ, Genitiv
- 🖨️ **Print-friendly** — clean layout when printed or saved as PDF
- 📱 **Responsive** — phone, tablet and desktop

## Deploy (GitHub Pages)

The site is served from the repository root on the `main` branch. Any push to `main` updates the live page. To (re)enable Pages: **Settings → Pages → Build and deployment → Source: “Deploy from a branch” → `main` / `/ (root)`**.
